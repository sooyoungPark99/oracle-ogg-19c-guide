## 3. Extract 설정
> orcl DB의 변경 데이터를 캡처하는 Extract 프로세스를 설정하는 과정

#### 파라미터 파일 설정 (EDIT PARAMS EXT1)
```
EXTRACT EXT1              ← 프로세스 이름
USERID oggadmin@orcl      ← orcl DB에 oggadmin으로 접속
EXTTRAIL ./dirdat/lt      ← 캡처한 변경 데이터를 lt 파일에 저장
TABLE scott.*;            ← scott 스키마의 모든 테이블 대상
```

```
GGSCI> EDIT PARAMS EXT1
```

```
EXTRACT EXT1
USERID oggadmin@orcl, PASSWORD oracle
EXTTRAIL ./dirdat/lt
TABLE scott.*;
```

```
# GGSCI가 orcl DB에 로그인
GGSCI> DBLOGIN USERID oggadmin@orcl PASSWORD oracle
-- Successfully logged into database. 이면 정상 (WARNING OGG-25108은 무시)

# scott 스키마의 DML 변경 내역을 Redo Log에 상세 기록하도록 설정
GGSCI> ADD SCHEMATRANDATA scott
-- INFO OGG-01788 이면 정상

# Redo Log에서 변경 데이터를 읽는 Extract 등록
GGSCI> ADD EXTRACT EXT1, TRANLOG, BEGIN NOW

# 캡처한 데이터를 저장할 Trail 파일 경로 등록
GGSCI> ADD EXTTRAIL ./dirdat/lt, EXTRACT EXT1, MEGABYTES 500

# Extract 프로세스 시작
GGSCI> START EXTRACT EXT1
GGSCI> INFO EXTRACT EXT1
-- Status: RUNNING 이면 정상

```

## 4. Data Pump 설정
> Extract가 저장한 Trail 파일을 clonedb 서버로 전달하는 역할

#### 파라미터 파일 설정
```
EXTRACT PMP1              ← 프로세스 이름 (Data Pump도 EXTRACT 타입)
USERID oggadmin@orcl      ← orcl DB 접속 정보
RMTHOST IP                ← 데이터를 보낼 대상 서버 IP
MGRPORT 7809              ← 대상 서버의 Manager 포트
RMTTRAIL ./dirdat/rt      ← 대상 서버에 저장할 Trail 파일명 (rt)
TABLE scott.*;            ← 전송 대상 테이블
```

```
GGSCI> EDIT PARAMS PMP1
```

```
EXTRACT PMP1
USERID oggadmin@orcl, PASSWORD oracle
RMTHOST 172.xx.xx.xx, MGRPORT 7809
RMTTRAIL ./dirdat/rt
TABLE scott.*;
```

```
# EXT1이 만든 lt 파일을 읽어서 처리하도록 등록
GGSCI> ADD EXTRACT PMP1, EXTTRAILSOURCE ./dirdat/lt

# 원격 서버에 rt 파일로 저장하도록 등록
GGSCI> ADD RMTTRAIL ./dirdat/rt, EXTRACT PMP1, MEGABYTES 500

# Data Pump 시작
GGSCI> START EXTRACT PMP1
GGSCI> INFO EXTRACT PMP1
-- Status: RUNNING 이면 정상

```

## 5. Replicat 설정

### 체크포인트 테이블 생성

```
GGSCI> DBLOGIN USERID oggadmin@clonedb PASSWORD oracle

GGSCI> ADD CHECKPOINTTABLE oggadmin.ggs_checkpoint
-- Successfully created checkpoint table 이면 정상
```
<img width="1363" height="227" alt="image" src="https://github.com/user-attachments/assets/bfed2fda-36a8-4e15-b1a8-0fe0a7dee642" />

### REP1 파라미터 파일 생성

> **ASSUMETARGETDEFS, CHECKPOINTTABLE**은 파라미터 파일에 넣지 않는다.

```
GGSCI> EDIT PARAMS REP1
```

```
REPLICAT REP1
USERID oggadmin@clonedb, PASSWORD oracle
MAP scott.*, TARGET scott.*;
```

### Replicat 등록 및 시작

```
GGSCI> ADD REPLICAT REP1, EXTTRAIL ./dirdat/rt, BEGIN NOW, CHECKPOINTTABLE oggadmin.ggs_checkpoint
GGSCI> START REPLICAT REP1
GGSCI> INFO ALL
```
