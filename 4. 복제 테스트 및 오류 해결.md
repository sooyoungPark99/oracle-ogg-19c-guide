# 4. 복제 테스트 및 오류 해결

---

## 1. 복제 테스트

```bash
# orcl에서 변경
sqlplus scott/tiger@orcl
```

```sql
UPDATE dept SET dname='SEOUL' WHERE deptno=10;
COMMIT;
EXIT;
```

```bash
# 10~20초 후 clonedb에서 확인
sqlplus scott/tiger@clonedb
```

```sql
SELECT * FROM dept WHERE deptno=10;
-- SEOUL 이 나오면 복제 성공
```

## 2. REP1 ABENDED 시 조치 순서

### 1) PK 제약 확인 (orcl, clonedb 양쪽)

```sql
-- PK 존재 여부 먼저 확인
select constraint_name, constraint_type
from user_constraints
where table_name in ('DEPT','EMP');

-- PK가 없는 경우에만 아래 실행 (이미 있으면 ORA-02260 오류 발생)
ALTER TABLE dept ADD CONSTRAINT pk_dept PRIMARY KEY (deptno);
ALTER TABLE emp  ADD CONSTRAINT pk_emp  PRIMARY KEY (empno);
```

### 2) OGG 재시작

```
GGSCI> STOP REPLICAT REP1
GGSCI> ALTER REPLICAT REP1, BEGIN NOW
GGSCI> START REPLICAT REP1
GGSCI> INFO REPLICAT REP1
-- Status: RUNNING 확인
```

### 3) orcl에서 DML 재수행 후 clonedb 반영 확인
 
```bash
# orcl에서 DML 수행
sqlplus scott/tiger@orcl
```
 
```sql
UPDATE dept SET dname='TEST' WHERE deptno=40;
COMMIT;
EXIT;
```
 
```bash
# 10~20초 후 clonedb에서 확인
sqlplus scott/tiger@clonedb
```
 
```sql
SELECT * FROM dept WHERE deptno=40;
-- TEST 가 나오면 복제 정상
EXIT;
```

## 3. 자주 발생하는 오류

| 오류 | 원인 | 해결 |
|------|------|------|
| `OGG-01790 ADD SCHEMATRANDATA 실패` | DBMS_XSTREAM_GG_ADM 권한 없음 | 2번 추가 권한 부여 재실행 |
| `OGG-02091 enable_goldengate_replication not true` | clonedb 파라미터 미설정 | clonedb에도 `ALTER SYSTEM SET ENABLE_GOLDENGATE_REPLICATION=TRUE` 설정 |
| `OGG-01296 Error mapping SCOTT.DEPT` | 양쪽 데이터 불일치 | orcl/clonedb 동일한 데이터로 맞춤 |
| `OGG-10144 CHECKPOINTTABLE not valid` | 파라미터 파일에 CHECKPOINTTABLE 넣음 | REP1 파라미터 파일에서 제거, ADD REPLICAT 시에만 사용 |
| `OGG-02760 ASSUMETARGETDEFS is ignored` | Trail에 이미 테이블 정의 포함 | REP1 파라미터 파일에서 ASSUMETARGETDEFS 제거 |
| REP1 ABENDED 반복 | 이전 trail 위치에서 오류 발생 | `ALTER REPLICAT REP1, BEGIN NOW`로 체크포인트 리셋 |

## 4. 실습 — DDL 미복제 확인

GoldenGate 기본 설정은 DML만 복제하고 DDL은 복제하지 않는다.

### orcl에서 emp 테이블 DROP

```sql
sqlplus scott/tiger@orcl
DROP TABLE emp;
COMMIT;
EXIT;
```

### clonedb에서 emp 테이블 확인

```sql
sqlplus scott/tiger@clonedb
SELECT COUNT(*) FROM emp;
-- 테이블이 그대로 존재하면 DDL 미복제 확인 완료
EXIT;
```

> orcl에서 emp가 삭제됐으므로 이후 테스트를 위해 3번 문서의 emp 생성 스크립트를 재수행한다.

### DDL 설정 관련 주의사항

DDL 패키지(`GGS_DDL_HIST` 테이블 등)가 설치되지 않은 환경에서는 DDL 관련 키워드 자체를 사용할 수 없다.

| 설정 | DDL 복제 여부 | 사용 가능 | 비고 |
|------|-------------|---------|------|
| DDL 설정 없음 (기본값) | ❌ DDL 복제 안 됨 | ✅ | **정상 동작** |
| `DDL EXCLUDE ALL` 추가 | - | ❌ | GGS_DDL_HIST 테이블 없어서 EXT1 ABEND |
| `DDL INCLUDE ALL` 추가 | ✅ DDL도 복제됨 | ❌ | 동일 이유로 ABEND |

> 결론: DDL 패키지 미설치 환경에서는 파라미터에 DDL 관련 설정을 아예 넣지 않는다.
